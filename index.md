# Архивирование в Linux

В Linux архивирование — это очень важный навык. Рано или поздно вам понадобится упаковать папку с проектом, сделать резервную копию, перенести сайт на другой сервер или просто собрать много файлов в один удобный файл.

На первый взгляд всё просто: есть папка, надо сделать архив. Но в Linux есть несколько разных инструментов, и они решают немного разные задачи.

Самые частые инструменты:

- `zip`
- `tar`
- `gzip`
- `xz`
- `zstd`
- `pv`

Разберёмся спокойно.

### zip

`zip` — это привычный формат архивов, который хорошо открывается почти везде: в Linux, Windows, macOS.

Пример:

```bash
zip -r azimut74.zip azimut74
```

Здесь:

- `zip` — программа архивирования
- `-r` — recursive, то есть рекурсивно пройти по всей папке
- `azimut74.zip` — имя будущего архива
- `azimut74` — папка, которую архивируем

##### Важный момент

Команда:

```bash
zip azimut74.zip azimut74
```

для папки обычно не подходит, потому что без `-r` программа не упакует всё содержимое папки.

Правильно:

```bash
zip -r azimut74.zip azimut74
```

Если не хочется видеть огромный список файлов в терминале:

```bash
zip -rq azimut74.zip azimut74
```

Ключ `-q` означает `quiet`, то есть тихий режим.

##### Когда использовать zip

`zip` удобно использовать, когда:

- архив нужно передать человеку на Windows;
- нужна максимальная совместимость;
- архив не очень большой;
- не особенно важны Linux-права, владельцы файлов и другие системные детали.

### tar

`tar` — это классический Linux-инструмент для упаковки файлов.

Сам по себе `tar` не обязательно сжимает данные. Его главная задача — взять много файлов и папок и собрать их в один файл.

Пример без сжатия:

```bash
tar -cf azimut74.tar azimut74
```

Здесь:

- `tar` — программа;
- `-c` — create, создать архив;
- `-f` — file, указать имя файла архива;
- `azimut74.tar` — имя архива;
- `azimut74` — папка, которую архивируем.

Такой архив может быть почти такого же размера, как исходная папка, потому что он просто собирает файлы в один контейнер.

Распаковка:

```bash
tar -xf azimut74.tar
```

Здесь:

- `-x` — extract, извлечь;
- `-f` — файл архива.

### tar + gzip

Очень часто встречается формат `.tar.gz`.

Это значит:

1. сначала `tar` собрал файлы в один поток;
2. потом `gzip` сжал этот поток.

Создать архив:

```bash
tar -czf azimut74.tar.gz azimut74
```

Распаковать:

```bash
tar -xzf azimut74.tar.gz
```

Здесь добавляется ключ:

```bash
-z
```

Он означает: использовать `gzip`.

`gzip` — старый, надёжный, совместимый, но не самый быстрый и не всегда самый эффективный вариант.

### tar + xz

Формат `.tar.xz` обычно сжимает сильнее, чем `gzip`, но может работать заметно медленнее.

Создать:

```bash
tar -cJf azimut74.tar.xz azimut74
```

Распаковать:

```bash
tar -xJf azimut74.tar.xz
```

Ключ:

```bash
-J
```

означает: использовать `xz`.

##### Когда использовать xz

`xz` удобно использовать, когда:

- хочется получить архив поменьше;
- скорость не очень важна;
- архив создаётся для долгого хранения.

Например, для финальных архивов, которые не нужно часто перепаковывать.

### tar + zstd

`zstd` — современный алгоритм сжатия. Он часто очень удачно балансирует между скоростью и размером архива.

Для больших папок с огромным количеством файлов это часто один из лучших вариантов.

Создать архив:

```bash
tar --zstd -cf azimut74.tar.zst azimut74
```

Распаковать:

```bash
tar --zstd -xf azimut74.tar.zst
```

Формат обычно называется:

```text
.tar.zst
```

То есть:

- `.tar` — файлы собраны в один архив;
- `.zst` — архив сжат через `zstd`.

Для большой Linux-папки чаще всего удобно использовать именно `tar + zstd`.

### pv

`pv` — это маленькая, но очень полезная программа для отображения прогресса.

Название можно запомнить как `pipe viewer` — просмотрщик потока.

Установить:

```bash
sudo apt install pv
```

Пример без сжатия:

```bash
tar -cf - azimut74 | pv > azimut74.tar
```

Тут появляется важная Linux-механика.

```bash
tar -cf - azimut74
```

Минус `-` вместо имени файла означает: не записывай архив сразу в файл, а выведи его в стандартный поток вывода.

Дальше:

```bash
| pv
```

Мы передаём поток через `pv`, чтобы видеть прогресс.

А потом:

```bash
> azimut74.tar
```

Записываем результат в файл.

### tar + pv + zstd

Вот хороший вариант для больших папок:

```bash
tar -cf - azimut74 | pv | zstd -T0 -o azimut74.tar.zst
```

Разберём команду:

```bash
tar -cf - azimut74
```

Собираем папку `azimut74` в `tar`-поток.

```bash
| pv
```

Показываем прогресс прохождения данных.

```bash
| zstd -T0 -o azimut74.tar.zst
```

Сжимаем поток через `zstd` и записываем в файл `azimut74.tar.zst`.

Ключ:

```bash
-T0
```

означает: использовать все доступные потоки процессора.

Распаковка такого архива:

```bash
zstd -d -c azimut74.tar.zst | tar -xf -
```

Здесь:

- `zstd -d` — распаковать;
- `-c` — вывести результат в поток;
- `| tar -xf -` — передать поток в `tar` и извлечь файлы.

### tar, а не ar

Иногда можно случайно написать:

```bash
ar -cf - azimut74 | pv | zstd -T0 -o azimut74.tar.zst
```

Но для нашей задачи это неправильно.

Правильно:

```bash
tar -cf - azimut74 | pv | zstd -T0 -o azimut74.tar.zst
```

`ar` — это другой инструмент. Он используется в основном для создания статических библиотек и других специальных архивов в Unix-системах.

Для папок, проектов и бэкапов нам нужен именно `tar`.

### .bashrc и .bash_aliases

В Linux shell — это программа, которая принимает ваши команды в терминале.

Чаще всего в Ubuntu, Pop!_OS и похожих системах используется Bash.

Когда Bash запускается, он читает специальные файлы настроек в домашней папке пользователя.

Один из главных файлов:

```bash
~/.bashrc
```

Символ `~` означает домашнюю папку текущего пользователя.

То есть это примерно:

```text
/home/username/.bashrc
```

Файл `.bashrc` выполняется каждый раз, когда вы открываете новый терминал.

Туда можно добавлять:

- переменные окружения;
- настройки терминала;
- псевдонимы команд;
- функции;
- подключение дополнительных файлов.

Но чтобы не превращать `.bashrc` в большую свалку, часто используют отдельный файл:

```bash
~/.bash_aliases
```

В него удобно складывать свои алиасы и небольшие функции.

В Ubuntu и Pop!_OS обычно в `.bashrc` уже есть блок, который автоматически подключает `.bash_aliases`, если он существует:

```bash
if [ -f ~/.bash_aliases ]; then
    . ~/.bash_aliases
fi
```

Это означает: если файл `~/.bash_aliases` существует, подключить его.

### Alias и функция

Обычный alias хорошо подходит для простых сокращений.

Например:

```bash
alias ll='ls -la'
```

После этого можно писать:

```bash
ll
```

вместо:

```bash
ls -la
```

Но alias плохо подходит для команд с аргументами.

Например, мы хотим писать:

```bash
flat azimut74
```

Чтобы это превращалось в:

```bash
tar -cf - azimut74 | pv | zstd -T0 -o azimut74.tar.zst
```

Здесь `azimut74` — это аргумент. Обычный alias с таким работает неудобно.

Поэтому лучше использовать bash-функцию.

### Делаем команды flat и unflat через .bash_aliases

Сначала убедимся, что установлены нужные программы:

```bash
sudo apt update
sudo apt install zstd pv
```

Теперь открываем файл `.bash_aliases`:

```bash
nano ~/.bash_aliases
```

Если файла ещё нет — он создастся.

Добавляем туда функцию для архивирования:

```bash
flat() {
    if [ -z "$1" ]; then
        echo "Использование: flat имя_папки_или_файла"
        return 1
    fi

    local source_path="$1"
    local archive_name="${source_path%/}.tar.zst"

    tar -cf - "$source_path" | pv | zstd -T0 -o "$archive_name"
}
```

Теперь добавим функцию для распаковки:

```bash
unflat() {
    if [ -z "$1" ]; then
        echo "Использование: unflat имя_архива.tar.zst"
        return 1
    fi

    local archive_path="$1"

    zstd -d -c "$archive_path" | tar -xf -
}
```

##### Полный блок для ~/.bash_aliases

```bash
flat() {
    if [ -z "$1" ]; then
        echo "Использование: flat имя_папки_или_файла"
        return 1
    fi

    local source_path="$1"
    local archive_name="${source_path%/}.tar.zst"

    tar -cf - "$source_path" | pv | zstd -T0 -o "$archive_name"
}

unflat() {
    if [ -z "$1" ]; then
        echo "Использование: unflat имя_архива.tar.zst"
        return 1
    fi

    local archive_path="$1"

    zstd -d -c "$archive_path" | tar -xf -
}
```

Сохраняем файл в `nano`:

```text
Ctrl + O
Enter
Ctrl + X
```

Теперь нужно перечитать настройки терминала:

```bash
source ~/.bashrc
```

Или можно просто закрыть терминал и открыть заново.

##### Использование

Архивируем папку:

```bash
flat azimut74
```

В результате получится файл:

```text
azimut74.tar.zst
```

Распаковываем:

```bash
unflat azimut74.tar.zst
```

Теперь у нас есть две удобные команды:

```bash
flat
```

для упаковки и:

```bash
unflat
```

для распаковки.

### Более аккуратная версия flat и unflat

Есть один нюанс. Если архив с таким именем уже существует, `zstd` может спросить про перезапись или отказаться.

Можно сделать функцию чуть умнее.

##### Финальный рекомендуемый вариант для ~/.bash_aliases

```bash
flat() {
    if [ -z "$1" ]; then
        echo "Использование: flat имя_папки_или_файла"
        return 1
    fi

    local source_path="${1%/}"
    local archive_name="${source_path}.tar.zst"

    if [ ! -e "$source_path" ]; then
        echo "Ошибка: '$source_path' не найден"
        return 1
    fi

    if [ -e "$archive_name" ]; then
        echo "Ошибка: архив '$archive_name' уже существует"
        return 1
    fi

    tar -cf - "$source_path" | pv | zstd -T0 -o "$archive_name"
}

unflat() {
    if [ -z "$1" ]; then
        echo "Использование: unflat имя_архива.tar.zst"
        return 1
    fi

    local archive_path="$1"

    if [ ! -f "$archive_path" ]; then
        echo "Ошибка: архив '$archive_path' не найден"
        return 1
    fi

    zstd -d -c "$archive_path" | tar -xf -
}
```

После добавления применяем настройки:

```bash
source ~/.bashrc
```

Проверяем:

```bash
flat azimut74
```

И потом:

```bash
unflat azimut74.tar.zst
```

### Вариант через sh-скрипты

Можно сделать отдельные скрипты в домашней папке пользователя.

Это уже похоже не просто на алиасы, а на маленькие собственные программы.

##### Создаём папку для личных скриптов

```bash
mkdir -p ~/bin
```

##### Создаём скрипт flat

```bash
nano ~/bin/flat
```

Вставляем:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
    echo "Использование: flat имя_папки_или_файла"
    exit 1
fi

source_path="${1%/}"
archive_name="${source_path}.tar.zst"

if [ ! -e "$source_path" ]; then
    echo "Ошибка: '$source_path' не найден"
    exit 1
fi

if [ -e "$archive_name" ]; then
    echo "Ошибка: архив '$archive_name' уже существует"
    exit 1
fi

tar -cf - "$source_path" | pv | zstd -T0 -o "$archive_name"
```

Делаем файл исполняемым:

```bash
chmod +x ~/bin/flat
```

##### Создаём скрипт unflat

```bash
nano ~/bin/unflat
```

Вставляем:

```bash
#!/usr/bin/env bash

if [ -z "$1" ]; then
    echo "Использование: unflat имя_архива.tar.zst"
    exit 1
fi

archive_path="$1"

if [ ! -f "$archive_path" ]; then
    echo "Ошибка: архив '$archive_path' не найден"
    exit 1
fi

zstd -d -c "$archive_path" | tar -xf -
```

Делаем файл исполняемым:

```bash
chmod +x ~/bin/unflat
```

##### Добавляем ~/bin в PATH

Теперь нужно убедиться, что папка `~/bin` есть в `PATH`.

Открываем:

```bash
nano ~/.bashrc
```

Добавляем в конец:

```bash
export PATH="$HOME/bin:$PATH"
```

Применяем:

```bash
source ~/.bashrc
```

Теперь команды будут работать так же:

```bash
flat azimut74
```

```bash
unflat azimut74.tar.zst
```

### Что выбрать: функции или sh-скрипты

Для обучения и личного использования проще начать с `~/.bash_aliases`.

Это быстро, удобно и понятно.

Для более серьёзного использования лучше `~/bin/flat` и `~/bin/unflat`.

Так команды становятся похожи на настоящие маленькие программы.

### Итог

Для Windows-совместимых архивов используйте:

```bash
zip -r archive.zip folder
```

Для Linux-бэкапов используйте:

```bash
tar
```

Для современных быстрых архивов удобно использовать:

```text
tar + zstd
```

Для отображения прогресса:

```bash
pv
```

А для удобства можно создать свои команды:

```bash
flat azimut74
```

и:

```bash
unflat azimut74.tar.zst
```

И это уже маленький шаг от простого пользователя Linux к человеку, который начинает мягко настраивать систему под себя. Именно так Linux постепенно становится не чужой операционной системой, а своей рабочей мастерской.
